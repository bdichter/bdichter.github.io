Fully Automated Hugo Publishing

July 30, 2022

After my friend told me all about Hugo I decided to create a quickstart website with content published to GitHub pages, automatically using GitHub Actions. Initially I installed Hugo using Brew locally, creating a repo and pushing to a repo. Then I finally decided to read about setting up GitHub pages. The main requirement is setting up a repo with the proper naming convention: user/organization.github.io. The details regarding the gh-pages branch and GitHub Actons is well-documented here by Hugo. Anytime I merge changes to the Main branch, GitHub Action will build the Hugo site and publish the public content to the gh-pages branch.
